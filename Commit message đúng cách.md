# Mục đích của commit message
Một commit message phải trả lời được 2 câu hỏi: bạn làm gì? và tại sao?

# Commit như thế nào ?

Cú pháp cơ bản: <type>: <subject>

type: Là loại thay đổi.:
- feat: Thêm tính năng mới.
- fix: Sửa 1 cái bug.
- docs: Sửa tài liệu.
- refactor: Sửa code (tối ưu,...) nhưng không thêm tính năng hay sửa bug.
- style: Sửa format code (linh tinh dấu chấm phẩy, thụt lề...).

subject: Là tiêu đề, mô tả ngắn gọn (dưới 50 chữ) cái bạn đã làm. Viết kiểu câu mệnh lệnh (ra lệnh) ở thì hiện tại.
